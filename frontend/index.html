How This Miner Works

Real backend pool mining

The Python backend connects to your chosen pool over Stratum, receives real jobs, and hashes them with actual Scrypt PoW (N=1024, r=1, p=1 — the same function Dogecoin uses; verified in tests against the Dogecoin genesis block). Shares that meet the pool target are submitted; accepted/rejected counters come exclusively from pool responses.

CPU vs GPU mode

CPU mode runs worker threads with hashlib.scrypt (the GIL is released, so threads scale across cores). GPU mode compiles an OpenCL scrypt kernel (NVIDIA/AMD/Intel, Windows/macOS/Linux). The kernel must pass a self-test against the CPU implementation before it's used, and every GPU share candidate is re-verified on the CPU before submission. If no usable GPU exists, it falls back to CPU threads and says so.

Pools & payouts

The default pool (zpool) needs no registration — your wallet address is the login and payouts go straight to it (subject to the pool's minimum payout rules). Registered pools (AikaPool, litecoinpool, F2Pool) need a free account; use account.worker as the username. Realistic expectations: CPU/GPU scrypt hashrates are tiny compared to ASICs, so actual DOGE payouts will be very small and slow to reach pool minimums.

Blockchain data

Block height, difficulty, network hashrate, price, recent blocks and address lookups come live from public explorers (Blockbook, Blockchair, BlockCypher, CoinGecko) — no API keys. The backend caches every response so your browser never hits provider rate limits.

What's real (everything visible)

Hashrate, hashes, nonces, share counters, pool status, difficulty/target, best share, CPU/MEM (psutil), GPU utilization (nvidia-smi), wallet balance (on-chain), price (live). Nothing on this page is simulated.
